# v0.20.0 — file output card with user-triggered download

When the agent produces a file you now see it, choose where to save it, and nothing lands in your Downloads folder without you deciding so.

## Files appear as a card in the side panel

The old `save_to_downloads` tool — which silently dropped files into your Downloads folder — is replaced by `output_file`. When the agent creates a file (CSV, JSON, Markdown, text, or any other format), a **download card** appears in the chat:

- **Filename · type · size** are shown at a glance.
- Click **Download** to save. The browser opens a **Save As dialog** so you pick the exact location — no more hunting in Downloads for what the agent created.

## Card availability and "expired" state

The file content lives in memory while the service worker is active. If the service worker goes idle or restarts (e.g. after the panel is closed for a while), the card shows **"expired"**. Just ask the agent to regenerate the file — the conversation history is intact, so it can reproduce it immediately.

## 中文摘要

agent 生成文件时你现在会在侧边栏看到一张下载卡片,由你决定保存位置——不再静默写入 Downloads 文件夹。

### 文件以卡片形式显示在侧边栏

旧的 `save_to_downloads` 工具(会悄悄把文件扔进 Downloads)已替换为 `output_file`。当 agent 生成文件(CSV、JSON、Markdown、纯文本等任意格式)时,聊天里会出现一张**下载卡片**:

- 一眼看到**文件名 · 类型 · 大小**。
- 点击 **Download** 保存。浏览器弹出 **另存为对话框**,由你选择保存位置——不再需要去 Downloads 里翻找。

### 卡片有效期与"已过期"状态

文件内容保存在内存中,在 Service Worker 活跃期间有效。如果 Service Worker 空闲超时或重启(例如侧边栏关闭一段时间后),卡片会显示 **"expired(已过期)"**。只需让 agent 重新生成——对话历史完整保留,agent 可以立即重新输出。
