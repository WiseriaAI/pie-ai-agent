# v0.19.4 — StepFun provider + smoother custom-provider setup

A small feature release: a new built-in LLM provider and two quality-of-life fixes for configuring custom providers.

## New provider: StepFun (阶跃星辰)

Pie now ships **StepFun** as a built-in provider (the 11th). It speaks the Anthropic wire at `https://api.stepfun.com/v1/messages` with Bearer auth, so it rides the same official-SDK transport as Anthropic / DeepSeek / MiniMax / MiMo.

Two models are curated:

| Model | Highlights | Context | Image input |
|---|---|---|---|
| `step-3.7-flash` | Native multimodal reasoning flagship | 256K | ✅ |
| `step-3.5-flash` | Reasoning + strong tool-calling, long-horizon tasks | 256K | — |

Pick **StepFun(阶跃星辰)** when creating an instance, paste your key, and choose a model — no base URL to configure.

## Custom providers: add models from the instance config

You can now add a model id to a custom provider directly from the instance configuration screen, instead of having to edit the provider first. The custom-provider **Base URL** field also now notes that it must be an **OpenAI-compatible** endpoint, so the requirement is clear before you paste (#119).
