# v0.12.0 — Web search, context ring, agent self-correction

The biggest minor release since v0.11. Four independent feature areas land together:

- **Web search** — a new `search_web` tool backed by Tavily (BYOK), with drill-down into the top results via the existing browser-tab tools.
- **Context ring** — a small composer indicator that shows how much of the model's context window the current session has consumed (Issue #59).
- **Agent self-correction** — deterministic loop / oscillation detection and an in-loop reflection mechanism that stops the ReAct loop from burning steps when it gets stuck (Issues #61, #64).
- **Model-authored give-up summary** — when the agent does have to abandon a task, it now writes its own 1–3 sentence "here's what I tried and why it didn't work" instead of a generic line (Issue #65).

## Web search (`search_web` tool)

A new built-in tool exposes web search to the agent through Tavily — a search engine tuned for AI agents that returns title / URL / snippet for each result.

### What's new

- **`search_web({query, max_results?})`** — calls Tavily, returns ranked results wrapped in a `<untrusted_search_result>` block. `max_results` clamps to 1–10 (default 5). Calls execute directly without a confirm card; the user pays per call via their own Tavily key.
- **Settings → Search** — a new 3rd segment in Settings hosts a 3-state UI (empty / configured / editing) for managing the Tavily key. The key is AES-GCM encrypted at rest in `chrome.storage.local`; the verified pill only shows when a successful `Save & test` actually verified the key.
- **System-prompt guidance** — the agent is taught to read snippets, pick 1–3 promising URLs, drill into them with `open_url` + `get_tab_content`, and synthesize across sources. Default disposition: one search → drill 2–3 → synthesize. Prefer drilling over re-searching.
- **Untrusted-data discipline** — search results live inside a new `untrusted_search_result` wrapper kind, and the system prompt explicitly tells the model never to follow instructions found in result text. The wrapper is wired into both `escapeUntrustedWrappers` and the dual-list lock-step `sanitizeText` chain in `snapshot.ts`.
- **BYOK key handling** — the key never leaves `chrome.storage.local` except on outbound HTTPS to `api.tavily.com`. New `host_permissions` entry: `https://api.tavily.com/*`.

### When to use

The agent decides automatically — but in practice: knowledge questions the current pinned tab can't answer, fact-checks against external sources, or any explicit "research this" request. Conversational questions and answers already in the current tab keep using the existing tools.

### Setup

Settings → Search → Get a key from https://tavily.com/ (1,000 free searches per month) → Add key → Save & test. Done.

## Context ring (#59)

A small ring icon next to the composer's send button shows how full the model's context window is, with a popover for details and color thresholds as utilization climbs. Built on a new `slot.usage` field that's persisted per session and rehydrated on session switch, so the indicator stays accurate across SW restarts.

### What's new

- New `SessionAgentState.contextUsage` + `AgentUsageMessage` wire event
- Per-step `done.usage` capture, persisted in session state
- Tombstone carries `contextUsage` across task boundaries (the ring doesn't reset to zero just because a task finished)
- Snapshot merge regression guarded by cross-layer e2e test
- Light/dark theme + i18n; popover closes on click-outside

## Agent self-correction (#61)

The ReAct loop stops burning steps when it gets stuck, and stops paying to re-send page snapshots it no longer needs. Three deterministic, zero-extra-LLM-cost improvements:

### Loop detection

Each step is fingerprinted by its tool calls (name + stable-sorted args) and kept in a small ring buffer. Two detectors fire:

- *Exact repeat* — the same action issued 3 times in a row (e.g. clicking the same dead button).
- *Repeat + error* — the same action that keeps erroring, caught after just 2 tries.

### Intra-episode reflection

When a loop is detected, the agent does **not** re-run the repeated action. Instead it inserts trusted self-correction guidance into the next turn (a `<reflections>` note: re-read the page, try a different element/tool, or `fail` cleanly) and shows a distinct **reflect** step in the trace. After 2 reflections without escape, the task stops with a clear message instead of grinding to the 30-step ceiling.

### Stale-snapshot elision

Only the most recent page snapshot is sent to the model with its full interactive-element list. Older snapshots keep their cheap semantic header (URL / title / headings / alerts / status) but their bulky element lists are replaced with a short marker — the agent always acts on the *current* page, so historical DOM dumps were just taxing the context window. Deterministic, zero-LLM-cost trim applied only on the wire; at-rest history is unchanged.

## Smarter loop detection & give-up summary (#64, #65)

### Oscillation detection (#64)

The loop detector now also catches **cycles** — the agent bouncing between the same N actions with no progress (`a→b→a→b`, or `a→b→c→a→b→c`). Routed through the same reflection mechanism. Still zero extra LLM calls, still deterministic; the ring buffer is widened to hold a full period-3 cycle. Periods up to 3 are scanned by default.

### Model-authored failure summary (#65)

When the agent exhausts its reflection budget and is force-stopped, it no longer ends with the generic "got stuck repeating the same action." Instead it gets one final **tools-disabled** turn to write its own 1–3 sentence summary — so the result reads like *"I kept clicking Place Order but no confirmation ever appeared, so the order couldn't be completed."* Hard termination guarantee unchanged: if that final turn errors, is aborted, or returns nothing, it falls back to the deterministic message. Costs exactly one extra LLM call, only on the rare give-up path.

## Notes & known tuning

- Tavily key is BYOK — `search_web` returns a clear "not configured" error when the key is missing, directing the user to Settings → Search; the agent surfaces this verbatim rather than trying to work around it.
- The exact-repeat threshold (3) is a fail-fast heuristic. Legitimately repeating an idempotent read (e.g. polling `get_tab_content` while a slow page loads) can trip a reflection; recoverable, and the threshold is a single tuned constant.
- Oscillation reflections obey the same `MAX_REFLECTIONS` cap as repeat detection — they don't extend how long a stuck task runs.
- The give-up summary is delivered as the task's final result (not streamed mid-turn) and is persisted as untrusted model text (same escaping as any agent output).
- `<reflections>` is documented in the system prompt as trusted runtime guidance (not page data); only the newest snapshot is shown in full — the agent should record anything it needs later in its reasoning or re-read with `get_tab_content`.

## Unchanged

- `MAX_STEPS = 30` remains the absolute backstop
- Prompt-injection `<untrusted_*>` wrapping unchanged elsewhere; only an additive new `untrusted_search_result` wrapper kind is introduced
- At-rest session history stays raw (resume after a service-worker restart is unaffected)
- New permission only: `host_permissions += https://api.tavily.com/*`. No other manifest change.
