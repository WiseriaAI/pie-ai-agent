# v0.12.0 — Agent self-correction (loop detection + reflection) & leaner context

The ReAct agent loop stops burning steps when it gets stuck, and stops paying to re-send page snapshots it no longer needs. Three independent, deterministic improvements land together (Issue #61).

## What changed

- **Loop detection (deterministic, zero extra LLM calls)**: each step is fingerprinted by its tool calls (name + stable-sorted args) and kept in a small ring buffer. Two detectors fire:
  - *Exact repeat* — the same action issued 3 times in a row (e.g. clicking the same dead button).
  - *Repeat + error* — the same action that keeps erroring, caught after just 2 tries.
- **Intra-episode reflection**: when a loop is detected, the agent does **not** re-run the repeated action. Instead it inserts trusted self-correction guidance into the next turn (a `<reflections>` note: re-read the page, try a different element/tool, or `fail` cleanly) and shows a distinct **reflect** step in the trace. After 2 reflections without escape, the task stops with a clear "got stuck repeating the same action" message instead of grinding to the 30-step ceiling.
- **Stale-snapshot elision**: only the most recent page snapshot is sent to the model with its full interactive-element list. Older snapshots keep their cheap semantic header (URL / title / headings / alerts / status) but their bulky element lists are replaced with a short marker — the agent always acts on the *current* page, so historical DOM dumps were just taxing the context window. This is a deterministic, zero-LLM-cost trim applied only on the wire (history at rest is unchanged), and it makes the existing token-budget guard rarely need to drop conversation history.

## Notes & known tuning

- The system prompt now tells the model that `<reflections>` is trusted runtime guidance (not page data) and that only the newest snapshot is shown in full — so it records anything it needs later in its reasoning or re-reads with `get_tab_content`.
- The exact-repeat threshold (3) is a fail-fast heuristic. Legitimately repeating an idempotent read (e.g. polling `get_tab_content` while a slow page loads) could trip a reflection; it is recoverable and the threshold is a single easily-tuned constant if false positives surface.

## Unchanged

- No manifest / permissions change — silent upgrade
- At-rest session history stays raw (resume after a service-worker restart is unaffected); prompt-injection `<untrusted_*>` wrapping is unchanged
- `MAX_STEPS = 30` remains the absolute backstop

## 中文摘要

ReAct agent loop 现在能自我修正，并顺手砍掉最肥的 context 浪费（Issue #61，三项独立改造）：

- **循环检测（确定性，0 额外 LLM 调用）**：每步按工具调用（名字 + 稳定排序的参数）生成签名，存进小 ring buffer。两档判定：*完全重复*——同一动作连续 3 次（反复点同一个死按钮）；*重复 + 错误*——同一动作反复报错，2 次即捕获。
- **任务内反思**：检测到循环时**不执行**该重复动作，而是把可信的自我修正指令注入下一轮（`<reflections>`：重读页面、换元素/工具、或干净地 `fail`），并在轨迹里显示一个独立的 **reflect** 步。连续 2 次反思仍无法脱困就以"卡在重复同一动作"明确终止，而不是一路烧到 30 步上限。
- **陈旧快照省略**：只有最近一张页面快照带完整交互元素列表发给模型；更早的快照保留廉价的语义头部（URL / 标题 / headings / alerts / status），巨型元素列表换成短 marker——agent 永远在**当前**页面决策，历史 DOM 纯属交税。这是只作用于发送层的确定性零成本裁剪（at-rest 历史不变），让既有 token-budget 几乎不再需要丢弃对话历史。

### 注意

- system prompt 现在明确告知模型：`<reflections>` 是可信的 runtime 指令（非页面数据），且只有最新快照是完整的——后续要用的信息请写进推理或用 `get_tab_content` 重新抓。
- 完全重复阈值（3）是 fail-fast 启发式：合法地重复某个幂等读取（如慢页加载时轮询 `get_tab_content`）可能触发一次反思，可恢复，且阈值是单一易调常量。

### 不变

- 无 manifest / 权限变更，静默升级
- at-rest session 历史保持 raw（SW 重启后 resume 不受影响）；`<untrusted_*>` 防注入 wrapper 不变
- `MAX_STEPS = 30` 仍是绝对兜底
