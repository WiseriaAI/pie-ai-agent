# v0.11.2 — Composer UI cleanup

The composer's action row tightens up. The three tool buttons (pick element / attach / record) collapse into a single `+` menu, send and stop become icon-only, the hint strip under the composer goes away, and dropdown shadows soften so they no longer punch through light-mode backgrounds.

## What changed

- **Tools menu**: pick / attach / record → single `+` button that opens a popover containing the same three actions, each with icon + label
- **Icon-only send**: text + `↵` glyph → borderless arrow-up-right icon
- **Icon-only stop**: warning-tinted dot + "Stop" → borderless outlined square icon
- **Composer layout**: bottom row is now `[+] ........ [instance · model] [send]` — the instance selector sits next to send instead of next to the tools menu
- **Lighter popover shadows**: `InstanceSelector`, `ToolsMenu`, `SkillSlashPopover`, and `PinnedTabDropdown` all switch to `0 8px 24px rgba(0,0,0,0.12)`, replacing the previous `shadow-lg` / inline 0.7-black combinations that read as solid black under light mode
- **Removed hint strip**: dropped the `/ skills` + `SHIFT ↵ NEWLINE` row beneath the composer; the placeholder already communicates the same affordances

## Unchanged

- No manifest / permissions change — silent upgrade
- Sessions, skills, and instances all preserved
